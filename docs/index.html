Ключевые возможности

⚡

Асинхронный парсинг

7 независимых воркеров собирают объявления каждые 3 минуты. При сбое одного — остальные продолжают работу.

🔁

Дедупликация

SHA-256 хеш(адрес + площадь + цена). Одно объявление на нескольких площадках → одна запись, поле seen_on[] хранит все источники.

🔔

Telegram-уведомления

Создайте подписку с фильтрами (город, цена, комнаты, источник). Бот пришлёт сообщение сразу при появлении нового объявления.

🗺️

Карта объявлений

Endpoint /listings/map возвращает координаты для Leaflet.js. Карта обновляется вместе с базой.

🧹

Авто-очистка

Снятые объявления помечаются inactive. Через 3 дня Cleanup Worker удаляет их из БД — каскадом из избранного.

📞

Парсинг телефонов

Асинхронный модуль извлекает номер телефона продавца через Avito Mobile API и сохраняет в карточке объявления.

🧩

Расширяемость

Новый источник = один новый файл parsers/mysource.py. Авторегистрация через metaclass. Нет изменений в остальном коде.

🔍

Полнотекстовый поиск

PostgreSQL tsvector + GIN-индекс. QueryBuilder по паттерну OCP — добавление фильтра = новый класс.

Логика дедупликации

1
Воркер получает объявление и вычисляет dedup_hash = sha256(address + area + price)
2
INSERT ... ON CONFLICT (dedup_hash) — атомарный upsert без локов на уровне приложения
3
Новое: вставка полной записи, source = источник-первооткрыватель, seen_on = [source]
4
Дубль: только seen_on обновляется через unnest + DISTINCT. Оригинальный source не меняется
5
PostgreSQL xmax = 0 trick: одним запросом определяем INSERT vs UPDATE без второго round-trip
